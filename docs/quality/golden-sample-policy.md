# AdMate Lens Golden Sample Policy

AdMate Lens uses golden samples to protect capture output quality before pixel diff automation is enforced. This policy defines which PNG and metadata files can become golden samples, how pending samples behave, and which artifacts must stay out of the repository.

## Scope

Golden QA covers generated capture outputs only after they are produced. It does not change capture rendering, media synthetic UI, creative injection, browser execution, or database schema.

Minimum first-run surfaces:

- `youtube-pc-instream-skip`
- `youtube-shorts`
- `demandgen-youtube-feed`
- `gdn-pc-display`
- `naver-smart-channel-mobile`
- `kakao-bizboard`

## Directory Contract

```text
tests/golden/
  manifests/   committed manifest files
  metadata/    committed placeholder or approved metadata fixtures
  images/      committed repo-safe approved golden PNGs only
  candidates/  generated current PNGs, ignored by git
  diffs/       generated diff PNGs, ignored by git
  reports/     generated JSON reports, ignored by git
```

Pending samples must not require local PNG files. A manifest with `sampleState: "pending-sample"` is a contract placeholder; dimension and pixel checks must skip file reads until an approved golden PNG is added.

## Manifest Rules

Each manifest must include:

- `schemaVersion`
- `surface`
- `channel`
- `productFamily`
- `productSurface`
- `sampleState`
- `sensitivity`
- `golden.imagePath`
- `golden.metadataPath`
- `golden.width`
- `golden.height`
- `candidate.imagePath`
- `candidate.outputDir`
- `diff.outputDir`
- `diff.maxPixelDiffRatio`
- `diff.maxMeanDelta`
- `diff.failOnDimensionMismatch`
- `requiredMetadata`
- `expectedMetadata`

Allowed `sampleState` values:

- `pending-sample`: no approved PNG yet; pixel diff is skipped.
- `approved`: repo-safe golden PNG exists and is checked locally.
- `external-only`: sensitive golden exists outside the repo; local pixel diff is skipped.

Allowed `sensitivity` values:

- `repo-safe-placeholder`
- `repo-safe`
- `external-sensitive`

## Metadata Rules

Common required metadata:

- `capturedAt`
- `durationMs`
- `resultCategory`
- `diagnostics`
- `diagnostics.captureQuality`
- `runtime.provider`
- `runtime.capturedAt`
- `runtime.durationMs`

Surface-specific required metadata:

| Surface | Required fields |
| --- | --- |
| `youtube-pc-instream-skip` | `youtubeAdType`, `instreamOpts.skipSeconds` |
| `youtube-shorts` | `youtubeAdType` |
| `demandgen-youtube-feed` | `productFamily`, `productSurface`, `youtubeAdType` |
| `gdn-pc-display` | `gdnViewportMode` |
| `naver-smart-channel-mobile` | `productFamily`, `productSurface` |
| `kakao-bizboard` | `productFamily`, `productSurface` |

Placeholder metadata may use synthetic timestamps and `runtime.provider: "placeholder"`, but must not include advertiser names, campaign names, real URLs, credentials, account identifiers, or private landing information.

## Golden PNG Selection

A PNG can be approved as golden only when:

- It is generated by AdMate Lens without manual image editing.
- The capture result is approved by an operator as report evidence quality.
- Metadata has `resultCategory: "ad_capture_ok"` or a documented approved review reason.
- Dimensions match the surface expectation.
- It contains no sensitive advertiser, campaign, account, URL, credential, or personal data if committed to the repo.
- The same request inputs can produce a comparable candidate capture.

Repo-safe examples should use neutral test creative, generic copy, and non-sensitive public context. Sensitive production captures must be stored externally.

## Candidate And Diff Rules

Candidate PNGs are generated current outputs used for comparison against an approved golden. Candidate files are never committed.

Diff outputs are generated artifacts:

- `tests/golden/diffs/{surface}/{runId}.diff.png`
- `tests/golden/reports/{surface}/{runId}.json`

Both are ignored by git. Reports should include surface, dimensions, changed pixel ratio, threshold, mean delta, paths, and failure reason.

## Threshold Defaults

| Surface | `maxPixelDiffRatio` | `maxMeanDelta` |
| --- | ---: | ---: |
| `youtube-pc-instream-skip` | `0.0075` | `3.0` |
| `youtube-shorts` | `0.0100` | `3.5` |
| `demandgen-youtube-feed` | `0.0100` | `3.5` |
| `gdn-pc-display` | `0.0200` | `5.0` |
| `naver-smart-channel-mobile` | `0.0050` | `2.5` |
| `kakao-bizboard` | `0.0050` | `2.5` |

Dimension mismatch is always a failure for `approved` local samples.

## Verification Commands

```bash
npm run check:golden-manifest
npm run check:golden-metadata
npm run check:golden-dimensions
npm run check:golden-pixels
npm run verify:golden
```

`verify:golden` is separate from `verify:harness` so ordinary checks remain lightweight. Pending samples pass manifest and metadata validation but skip local PNG and pixel diff checks.
