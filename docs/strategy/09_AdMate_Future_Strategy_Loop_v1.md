# AdMate Future Strategy Loop v1

작성일: 2026-04-30  
문서 상태: 초안 v1  
작성 목적: AdMate와 Hermes가 장기적으로 나스미디어의 광고 운영 지식, 내부 업무 데이터, 외부 시장/경쟁사 정보를 기반으로 신규 솔루션 기회를 발굴하고 PoC 기획/개발로 연결하는 미래 전략 루프를 정의한다.

---

## 1. Executive Summary

AdMate의 1차 목표는 미디어플래너의 반복 업무를 줄이고, 캠페인 사고를 예방하며, 광고 운영 전 과정을 AI Agent 기반으로 자동화하는 것이다.

그러나 AdMate와 Hermes가 장기적으로 축적하게 될 데이터는 단순 운영 로그가 아니다.

Hermes는 다음 정보를 지속적으로 축적하게 된다.

- 미디어플래너가 자주 겪는 업무 병목
- 반복되는 정책 질문
- 반복되는 캠페인 세팅 오류
- 자주 발생하는 운영 이상 신호
- 광고주가 자주 요청하는 보고/증빙 형태
- 업종/목표별 캠페인 성과 패턴
- 운영자의 판단과 예외 기준
- 현재 회사가 유료로 사용하는 외부 솔루션의 대체 가능성
- 경쟁사 또는 시장에서 등장하는 신규 솔루션

이 데이터가 충분히 쌓이면 AdMate/Hermes는 단순히 현재 업무를 자동화하는 수준을 넘어, **우리 회사가 앞으로 어떤 솔루션을 만들어야 하는지 제안하는 전략 엔진**으로 확장될 수 있다.

이를 본 문서에서는 다음 이름으로 정의한다.

```text
Business Opportunity Discovery & Build Loop
= 사업 기회 탐색 및 솔루션 구축 루프
```

이 루프는 AI가 마음대로 회사를 운영하거나 제품을 개발하는 구조가 아니다.

정확한 개념은 다음과 같다.

```text
AI가 시장·경쟁사·내부 업무 데이터를 분석해 솔루션 기회를 제안하고,
사람의 승인 하에 PoC 기획과 개발 작업으로 전환하는 장기 전략 루프
```

---

## 2. 왜 이 루프가 필요한가

AdMate가 운영되기 시작하면 회사 내부에 다음과 같은 데이터가 축적된다.

| 데이터 | 의미 |
|---|---|
| Compass 질문 이력 | 플래너가 어떤 정책/매체 이슈를 자주 묻는지 |
| Sentinel 검수 결과 | 어떤 캠페인 세팅 오류가 반복되는지 |
| Sentinel 운영 감지 | 어떤 캠페인 유형에서 이상 신호가 자주 발생하는지 |
| Lens 캡처 요청 | 어떤 매체/지면/보고 유형의 증빙 요청이 많은지 |
| Foresight 예측 오차 | 어떤 업종/목표에서 예측이 어렵거나 편차가 큰지 |
| Hermes 피드백 | 운영자가 실제로 어떤 판단을 내리는지 |
| Cost Center | 어떤 AI 기능이 비용 대비 효과적인지 |
| Weekly Intelligence | 외부 기술/경쟁사/시장 변화가 무엇인지 |

이 데이터는 단순한 운영 기록이 아니라, 회사의 업무 병목과 기회가 드러나는 신호다.

예를 들어 다음과 같은 질문이 가능해진다.

```text
미디어플래너들이 가장 많이 묻는 정책 영역은 무엇인가?
반복되는 세팅 오류는 어떤 캠페인 유형에서 많이 발생하는가?
광고주 보고 업무에서 가장 시간이 많이 드는 캡처 유형은 무엇인가?
우리가 현재 외부 유료 솔루션으로 해결하고 있는 업무 중 직접 만들 수 있는 것은 무엇인가?
경쟁사가 제공하지만 우리에게 없는 기능은 무엇인가?
내부 데이터로 차별화 가능한 신규 서비스는 무엇인가?
```

이 질문에 답할 수 있다면, AdMate는 단순 업무 자동화 플랫폼을 넘어 회사의 신규 솔루션 개발 방향을 제안하는 전략 자산이 된다.

## 3. 현재 계획에 포함해야 하는 수준

이 루프는 중요하지만, 현재 AdMate 1차 실행 범위에 깊게 포함하면 안 된다.

이유는 다음과 같다.

1. 현재 우선순위는 AdMate 제품군과 Agent Core의 구조 안정화다.
2. Business Opportunity Loop는 충분한 내부 데이터와 Hermes 학습이 쌓인 뒤 의미가 커진다.
3. 임원 보고에서 너무 크게 강조하면 과도한 자율 AI 개발처럼 보일 수 있다.
4. 보안, 권한, 승인, 책임 소재가 더 복잡해진다.
5. 현재는 Weekly Intelligence Upgrade Loop만 운영 루틴으로 우선 구축하는 것이 현실적이다.

따라서 현재 문서와 발표에서는 다음 수준으로 다룬다.

```text
현재 핵심 계획:
AdMate AI Agent 기반 광고 운영 자동화 플랫폼 구축

현재 운영 루틴:
LLM Cost Center + Weekly Intelligence Upgrade Loop

장기 비전:
Hermes에 축적된 내부 업무 지식과 외부 시장 정보를 결합해,
신규 솔루션 기회를 발굴하고 PoC 개발로 연결하는 전략 루프 확장
```

## 4. Business Opportunity Discovery & Build Loop 정의

## 4.1 정의

Business Opportunity Discovery & Build Loop는 Hermes에 축적된 내부 도메인 지식과 외부 시장/경쟁사 정보를 기반으로, 회사가 직접 구축할 가치가 있는 신규 솔루션 후보를 발굴하고 사업성을 평가한 뒤, 사람 승인 하에 PoC 기획과 개발로 연결하는 장기 전략 루프다.

## 4.2 사업성의 의미

여기서 사업성은 단순히 매출만 의미하지 않는다.

AdMate 관점에서 사업성은 다음 세 가지를 포함한다.

1. **직접 수익화 가능성**  
   광고주에게 별도 서비스나 고급 리포트, 컨설팅, 플랫폼 형태로 제공할 수 있는가?

2. **광고주 서비스 품질 개선 가능성**  
   직접 수익화가 아니더라도 광고주에게 더 빠르고 정확한 서비스를 제공해 경쟁력을 높일 수 있는가?

3. **외부 유료 플랫폼 대체 가능성**  
   현재 회사가 비용을 지불하고 사용하는 외부 솔루션을 내부 도구로 대체하거나 비용을 줄일 수 있는가?

추가로 다음 기준도 함께 본다.

- 내부 업무 시간 절감 가능성
- 데이터분석팀의 기술 자산화 가능성
- 기존 AdMate 플랫폼과의 연계 가능성
- 보안/ISMS 대응 가능성
- 유지보수 난이도
- PoC 구현 기간

## 5. 전체 루프 구조

Business Opportunity Discovery & Build Loop는 다음 흐름으로 운영된다.

```text
1. Hermes 도메인 지식 축적
2. 내부 업무 pain point mining
3. 외부 시장/경쟁사/기술 조사
4. 솔루션 기회 후보 발굴
5. 사업성 평가
6. 사람 승인
7. 기획안 작성
8. Codex/Paperclip Builder Agent 작업화
9. PoC 구축
10. 결과 평가
11. 정식 제품화 또는 폐기
12. 결과를 다시 Hermes에 학습
```

## 6. 주요 구성 Agent

이 루프는 여러 전문 Agent로 구성할 수 있다.

| Agent | 역할 |
|---|---|
| Strategy Direction Agent | CEO/CMO/팀장 지시를 전략 질문으로 변환 |
| Internal Pain Point Agent | AdMate 사용 로그에서 내부 업무 병목 탐색 |
| Market Intelligence Agent | 경쟁사/외부 솔루션/시장 동향 조사 |
| Technology Intelligence Agent | 최신 AI/MarTech 기술 적용 가능성 조사 |
| Business Case Agent | 사업성, ROI, 대체 가능성 평가 |
| Product Planning Agent | PoC 기획안, 요구사항, 범위 정의 |
| Builder Orchestration Agent | Codex/Paperclip 등 개발 Agent로 작업 이관 |
| Evaluation Agent | PoC 결과, 비용, 효과, 리스크 평가 |
| Governance Agent | 보안, 권한, ISMS, 승인 흐름 점검 |

이 Agent들은 독립 제품이 아니라, 장기적으로 AdMate Agent Core 위에서 동작하는 전략 레이어로 정의한다.

## 7. 사람 승인 기반 운영 원칙

이 루프에서 가장 중요한 원칙은 사람 승인이다.

나쁜 표현:

```text
AI가 알아서 우리 회사에 필요한 서비스를 찾아서 개발한다.
```

좋은 표현:

```text
AI가 내부 업무 데이터와 외부 시장 정보를 분석해 솔루션 기회를 제안하고,
사람의 승인 하에 PoC 기획과 개발 작업으로 전환한다.
```

## 7.1 승인 단계

| 단계 | 승인 주체 |
|---|---|
| 솔루션 후보 등록 | 데이터분석팀 리더/관리자 |
| 사업성 평가 진행 | 팀장 또는 지정 관리자 |
| PoC 개발 승인 | 의사결정권자 또는 프로젝트 오너 |
| 외부 데이터 사용 | 보안/권한 검토 필요 |
| 유료 API/모델 사용 | Cost Center 기준 승인 |
| 정식 제품화 | 임원/사업부 의사결정 |

## 7.2 자동화 가능한 부분과 아닌 부분

자동화 가능한 부분:

- 반복 질문/오류 패턴 분석
- 외부 솔루션 조사
- 기술 업데이트 요약
- 사업성 평가 초안 작성
- PRD 초안 작성
- Codex 작업 프롬프트 생성
- PoC 테스트 체크리스트 생성

사람 승인이 필요한 부분:

- 실제 개발 착수
- 예산 투입
- 외부 데이터 사용
- 유료 API 대량 사용
- 제품명/브랜드 확정
- 정식 출시 결정
- 조직 업무 프로세스 변경

## 9. 솔루션 후보 평가 프레임워크

각 후보는 다음 기준으로 평가한다.

| 평가 항목 | 설명 | 점수 예시 |
|---|---|---|
| 내부 pain point 강도 | 실제 업무에서 얼마나 자주/크게 발생하는 문제인가 | 1~5 |
| 광고주 가치 | 광고주에게 더 나은 서비스를 제공하는가 | 1~5 |
| 직접 수익화 가능성 | 별도 과금/서비스화 가능성이 있는가 | 1~5 |
| 외부 비용 대체 가능성 | 현재 유료 도구를 대체하거나 절감할 수 있는가 | 1~5 |
| 데이터 차별성 | 나스미디어 내부 데이터로 차별화 가능한가 | 1~5 |
| 구현 난이도 | PoC 구현 난이도는 낮은가 | 1~5 |
| 보안/ISMS 리스크 | 민감 데이터 리스크가 낮은가 | 1~5 |
| 기존 AdMate 연계성 | Compass/Sentinel/Lens/Foresight와 연결되는가 | 1~5 |
| 유지보수 가능성 | 운영 부담이 과도하지 않은가 | 1~5 |

## 9.1 우선순위 계산 예시

```text
Opportunity Score
= Pain Point 강도
+ 광고주 가치
+ 외부 비용 대체 가능성
+ 데이터 차별성
+ 기존 AdMate 연계성
- 구현 난이도
- 보안 리스크
```

정확한 수식은 추후 조정하되, 초기에는 정성 평가와 점수 평가를 병행한다.

## 10. 솔루션 후보 예시

아래는 향후 Hermes가 발굴할 수 있는 후보의 예시다.

## 10.1 광고주 보고 자동화 고도화

문제:

- 보고서 제작과 캡처 삽입에 시간이 많이 소요됨

가능 솔루션:

- Sentinel 운영 이력 + Lens 캡처 + Foresight 예측을 결합한 자동 보고서 생성

사업성:

- 내부 업무 시간 절감
- 광고주 서비스 품질 향상
- 고급 리포트 상품화 가능성

## 10.2 정책 리스크 사전 진단 솔루션

문제:

- 소재/랜딩/업종별 정책 리스크를 캠페인 집행 전 확인하는 데 시간이 걸림

가능 솔루션:

- Compass 정책 QA + Sentinel Pre-launch 검수 결합

사업성:

- 광고 심사 반려 감소
- 광고주 신뢰도 향상
- 사전 컨설팅 서비스화 가능성

## 10.3 외부 캡처 도구 대체

문제:

- 광고 게재 화면 캡처와 증빙 제작에 외부 도구 또는 수작업 의존

가능 솔루션:

- Lens 기반 매체별 캡처 자동화 고도화

사업성:

- 외부 비용 대체
- 보고 품질 표준화
- 내부 운영 효율 향상

## 10.4 업종별 성과 벤치마크 서비스

문제:

- 캠페인 제안 시 업종/목표별 예상 성과 기준이 분산됨

가능 솔루션:

- Foresight 기반 업종별 단가/성과 벤치마크 대시보드

사업성:

- 제안 품질 향상
- 광고주 컨설팅 품질 강화
- 장기적으로 유료 인사이트 서비스 가능

## 10.5 운영자 판단 기반 알림 최적화

문제:

- 이상 감지 알림이 너무 많으면 피로도가 높아짐

가능 솔루션:

- Hermes가 업종/목표/담당자별 false positive 기준을 학습해 알림 민감도 최적화

사업성:

- 운영 효율 향상
- 플래너 만족도 증가
- Sentinel 경쟁력 강화

## 13. Weekly Intelligence Loop와의 관계

Business Opportunity Loop는 Weekly Intelligence Loop보다 상위 개념이다.

| 구분 | Weekly Intelligence Loop | Business Opportunity Loop |
|---|---|---|
| 목적 | 최신 기술 변화 탐색 및 기존 플랫폼 고도화 | 신규 솔루션 기회 발굴 및 사업성 평가 |
| 주기 | 주간 | 월간/분기 또는 필요 시 |
| 초점 | AI/MarTech 기술, API, 모델, 비용 | 회사 경쟁력, 광고주 가치, 외부 솔루션 대체 |
| 산출물 | 기술 리포트, 적용 후보, 작업 backlog | 사업 기회 후보, PoC 기획안, 전략 제안 |
| 실행 범위 | 기존 AdMate 개선 | 신규 기능/제품/서비스 가능성 |

초기에는 Weekly Intelligence Loop를 먼저 운영하고, Hermes 데이터가 충분히 쌓이면 Business Opportunity Loop를 단계적으로 도입한다.

## 14. 도입 단계

## Phase 1. 문서화 및 장기 비전 포함

현재 단계에서는 Business Opportunity Loop를 장기 비전으로 문서화한다.

- 임원 보고 마지막 부분에 간략히 언급
- 현재 개발 범위에는 포함하지 않음
- AdMate의 장기 확장 가능성으로 제시

## Phase 2. 내부 Pain Point Mining

Hermes에 운영 데이터가 쌓이면 내부 업무 병목을 분석한다.

- 반복 정책 질문
- 반복 세팅 오류
- 반복 캡처 요청
- 반복 알림 오탐
- 예측 오차가 큰 업종/목표

## Phase 3. Market & Competitor Intelligence

외부 시장과 경쟁사 솔루션을 정기적으로 분석한다.

- 경쟁사 기능
- 외부 유료 도구
- 광고 운영 SaaS
- AI Agent/MarTech 트렌드

## Phase 4. Opportunity Scoring

내부 pain point와 외부 시장 정보를 결합해 후보를 점수화한다.

## Phase 5. PoC Planning & Build

승인된 후보를 PoC 기획안으로 만들고 Codex/Paperclip Builder Agent에게 작업을 이관한다.

## Phase 6. Evaluation & Productization

PoC 결과를 평가하고 정식 제품화 여부를 결정한다.

## 16. 리스크와 통제 방안

## 16.1 주요 리스크

| 리스크 | 설명 |
|---|---|
| 과도한 자동화 인식 | AI가 사람 의사결정을 대체하는 것처럼 보일 수 있음 |
| 잘못된 사업성 판단 | 불충분한 데이터로 잘못된 기회 평가 가능 |
| 보안 리스크 | 외부 리서치/LLM 사용 시 민감 정보 노출 가능 |
| 개발 범위 폭증 | 너무 많은 후보를 동시에 PoC로 진행할 위험 |
| 유지보수 부담 | 만든 솔루션이 운영 부담으로 돌아올 수 있음 |
| 조직 수용성 | 현업이 필요성을 느끼지 못하면 사용되지 않을 수 있음 |

## 16.2 통제 방안

- 사람 승인 기반 운영
- PoC 범위 제한
- 보안/ISMS 사전 검토
- Cost Center 예산 기준 적용
- 현업 pain point 검증
- 명확한 성공 기준 설정
- 폐기 기준 사전 정의
- 결과를 Hermes에 다시 학습

## 18. 최종 요약

Business Opportunity Discovery & Build Loop는 지금 당장 실행할 핵심 범위는 아니다.

하지만 AdMate와 Hermes의 장기 진화 방향으로 반드시 품고 가야 한다.

```text
현재 목표:
AdMate AI Agent 기반 광고 운영 자동화 플랫폼 구축

중기 목표:
운영 데이터와 피드백을 Hermes에 축적해 회사 고유의 광고 운영 지식 자산화

장기 목표:
Hermes의 도메인 지식과 외부 시장 정보를 결합해,
우리 회사가 직접 구축할 가치가 있는 신규 솔루션 기회를 발굴하고,
사람 승인 하에 PoC 개발과 제품화로 연결
```

이 루프가 정착되면 AdMate는 단순 업무 자동화 플랫폼을 넘어, 나스미디어의 미래 솔루션 전략을 제안하는 AI 기반 제품 전략 엔진으로 확장될 수 있다.
