# AdMate Media Planner Brief v1

작성일: 2026-04-30  
문서 상태: 초안 v1  
대상: 미디어플래너 / 캠페인 운영 실무자  
작성 목적: AdMate가 실제 미디어플래너의 업무를 어떻게 줄이고, 실수를 어떻게 막고, 캠페인 운영 품질을 어떻게 높이는지 실무 관점에서 설명하기 위한 문서. 향후 미디어플래너 대상 발표 슬라이드의 원본 자료로 활용한다.

---

## 1. 한 줄 정의

AdMate는 미디어플래너를 대체하는 시스템이 아니라, 반복 확인·정책 검색·세팅 검수·캡처·운영 이상 감지를 대신 처리해 플래너가 전략과 판단에 집중하도록 돕는 **AI 업무 파트너**다.

```text
AdMate
기획부터 운영까지 미디어플래너의 반복 업무를 줄여주는
AI Agent 기반 광고 운영 자동화 플랫폼
```

---

## 2. 미디어플래너가 매일 겪는 문제

미디어플래너의 업무는 단순히 캠페인을 세팅하고 운영하는 일이 아니다. 하나의 캠페인을 준비하고 운영하기 위해 정책 확인, 세팅 검수, 예산/기간/타겟 확인, 랜딩 URL 점검, 운영 중 모니터링, 캡처/보고, 다음 제안 준비가 반복된다.

| 업무 상황 | 현재 어려움 |
|---|---|
| 정책 확인 | 매체별 정책과 가이드가 흩어져 있어 직접 검색해야 함 |
| 캠페인 세팅 | 미디어믹스와 실제 매체 세팅값이 맞는지 수작업 확인 필요 |
| 예산/기간/타겟 검수 | 작은 숫자 단위 오류가 큰 사고로 이어질 수 있음 |
| 랜딩 URL 확인 | 오타, 404, 잘못된 페이지 연결 가능성 존재 |
| 운영 중 모니터링 | 예산 소진, CPC/CTR 변화, 캠페인 상태를 사람이 수시로 확인해야 함 |
| 캡처/보고 | 광고 게재 화면 캡처와 보고서용 증빙 이미지를 반복 생성해야 함 |
| 다음 제안 준비 | 과거 성과와 업종별 기준을 찾고 비교하는 데 시간이 걸림 |

이런 업무는 중요하지만 대부분 전략적 판단보다 반복 확인에 가깝다. AdMate는 이 반복 확인 업무를 줄이고, 플래너가 더 중요한 판단에 집중하도록 돕는다.

---

## 3. AdMate가 제공하는 캠페인 운영 사이클

AdMate는 캠페인의 전 과정을 하나의 흐름으로 연결한다.

```text
기획
→ 정책 확인
→ 캠페인 세팅 검수
→ 집행 시작
→ 운영 모니터링
→ 캡처/보고
→ 다음 캠페인 예측
```

| 단계 | 플랫폼 | 플래너가 얻는 도움 |
|---|---|---|
| 기획 | AdMate Foresight | 업종/목표/예산 기준 예상 성과 확인 |
| 정책 확인 | AdMate Compass | 매체 정책과 가이드를 빠르게 질문 |
| 사전 검수 | AdMate Sentinel | 미디어믹스와 실제 세팅값 오류 자동 확인 |
| 운영 감지 | AdMate Sentinel | 예산/KPI/상태 이상을 자동 감지 |
| 캡처/보고 | AdMate Lens | 광고 게재 화면과 증빙 이미지 자동 생성 |
| 학습 | AdMate Agent Core | 운영자 판단이 다음 기준에 반영 |

---

## 4. AdMate Compass: 정책 확인 시간을 줄이다

광고 정책은 매체마다 다르고 자주 바뀐다. Meta, Google, Naver, Kakao, X의 정책과 가이드를 직접 검색하고 확인하는 데 시간이 걸린다.

AdMate Compass는 광고 플랫폼 정책과 가이드의 방향을 잡아주는 Policy Intelligence Agent다.

플래너는 다음과 같이 질문할 수 있다.

```text
이 소재 Meta 정책상 문제 있을까?
건기식 광고에서 주의해야 할 문구 알려줘.
Google Ads에서 랜딩 페이지 관련 정책 확인해줘.
Naver 검색광고에서 병의원 업종 제한 사항 알려줘.
```

기대 효과:

- 정책 확인 시간 단축
- 특정 담당자 지식 의존도 감소
- 광고주 문의 대응 속도 향상
- 정책 리스크 사전 감소

```text
Before
매체별 도움말 검색 → 문서 탐색 → 담당자에게 재확인 → 답변 정리

After
AdMate에 질문 → 공식 가이드 기반 답변 확인 → 필요한 경우 근거 링크 확인
```

## 5. AdMate Sentinel: 캠페인 사고를 줄이다

AdMate Sentinel은 캠페인 시작 전과 시작 후를 모두 감시한다.

```text
Pre-launch Validation
= 시작 전 세팅 검수

Live Monitoring
= 시작 후 실시간 운영 감지
```

### 5.1 시작 전: 세팅 오류 검수

캠페인 세팅은 작은 실수가 큰 사고로 이어질 수 있다. 예산 단위 오류, 기간 오류, 타겟 오류, 랜딩 URL 오류, 캠페인 상태 오류, 미디어믹스와 실제 세팅값 불일치가 대표적이다.

Sentinel은 미디어믹스/엑셀의 캠페인 세팅 정보와 실제 매체 광고 플랫폼 내 세팅값을 비교한다.

확인 항목 예시:

- 예산
- 기간
- 타겟
- 캠페인 상태
- 랜딩 URL
- 캠페인명/광고세트명
- 구매 유형
- 캠페인 목표

기대 효과:

- 캠페인 시작 전 오류 발견
- 수작업 검수 시간 감소
- 사고 가능성 감소
- 검수 이력과 timestamp 자동 기록

### 5.2 시작 후: 실시간 모니터링과 알림

캠페인 시작 후에도 플래너는 예산 소진 속도, 예산 미소진, CTR 급락, CPC/CPM 급등, 캠페인 상태 변경을 계속 확인해야 한다.

Sentinel은 캠페인 운영 중 주요 이상 신호를 감지하고 Slack/Email로 알려준다. 운영자는 알림에 대해 확인 완료, 1시간 보류, 오늘 중지, 알림 종료, 알림 재개, 검토 필요로 대응할 수 있다.

이 대응 이력은 AdMate Agent Core에 기록되고, Hermes가 이후 유사 캠페인 판단 기준을 고도화하는 데 활용한다.

```text
Before
직접 매체 관리자 접속 → 수치 확인 → 이상 여부 판단 → 팀원 공유

After
이상 신호 자동 감지 → Slack 알림 확인 → 버튼으로 대응 → 이력 자동 기록
```

## 6. AdMate Lens: 캡처와 증빙 업무를 줄이다

광고 운영과 보고 과정에서는 광고 게재 화면, 상품, 지면, 소재 이미지 등 다양한 캡처가 필요하다. 하지만 캡처 업무는 반복적이고 시간이 많이 든다.

AdMate Lens는 광고 게재 화면과 보고서용 증빙 이미지를 자동 생성한다.

예시:

- YouTube 광고 게재 화면
- 모바일 인피드 광고 화면
- 네이티브 광고 UI
- 상품/브랜드 페이지 캡처
- 보고서용 고해상도 이미지

기대 효과:

- 반복 캡처 시간 감소
- 보고서 준비 속도 향상
- 이미지 품질 표준화
- 캠페인 기록과 캡처 자산 연결

```text
Before
직접 화면 구성 → 캡처 → 편집 → 보고서 삽입

After
캠페인/매체 정보 입력 → 자동 캡처 생성 → 보고서용 이미지 사용
```

---

## 7. AdMate Foresight: 다음 캠페인을 더 빠르게 예측하다

캠페인 제안이나 기획 단계에서 플래너는 과거 성과와 업종별 기준을 참고해야 한다. 그러나 유사 캠페인 검색, 업종/목표별 평균 CPM/CPC/CTR 기준 확인, 시즌/시장 상황 반영은 시간이 오래 걸린다.

AdMate Foresight는 과거 광고 데이터와 시장 트렌드를 기반으로 예상 성과를 제공한다. 1차 PoC는 Meta를 우선 대상으로 한다.

예상 제공 지표:

- CPM
- CPC
- CTR
- VTR
- CPV

입력 예시:

- 업종
- 캠페인 목표
- 예산
- 기간
- 국가
- 소재 유형

기대 효과:

- 기획 단계 의사결정 시간 단축
- 업종/목표별 단가 기준 내부 자산화
- 제안 품질 표준화
- 실제 운영 결과와 다음 기획의 연결

```text
Before
과거 사례 검색 → 엑셀 정리 → 담당자 경험 보정 → 예상 효율 산출

After
업종/목표/예산 입력 → 예상 지표 확인 → 제안 방향 수립
```

## 8. AdMate Agent Core: 내 판단이 다음 기준이 되다

AdMate의 중요한 차별점은 단순 자동화가 아니라 학습이다.

AdMate Agent Core는 Openclaw와 Hermes를 기반으로 한다.

- Openclaw: 조건과 스케줄에 따라 업무를 실행하는 자동화 실행 엔진
- Hermes: 사용자 이벤트와 운영자 판단을 학습해 지식을 축적하는 지능/메모리 엔진

운영자가 “이 캠페인은 CPC가 20% 올랐지만 해당 업종/목표에서는 정상 범위”라고 판단하면, 이 판단이 기록되고 Hermes가 유사 상황에서 더 나은 기준을 만들 수 있다.

즉, 플래너의 경험이 개인의 기억으로만 남지 않고 회사의 운영 지식으로 축적된다.

## 9. 실제 업무 시나리오

### 시나리오 1. 캠페인 집행 전 정책 확인

```text
상황: 건기식 캠페인 소재 문구가 정책상 가능한지 확인해야 함
기존 방식: 매체 도움말 검색 → 유사 문서 확인 → 담당자 문의 → 광고주 답변 정리
AdMate 사용: Compass에 질문 → 공식 가이드 기반 답변 확인 → 주의 문구 정리
효과: 정책 확인 시간 단축, 답변 근거 확보
```

### 시나리오 2. 캠페인 시작 전 세팅 검수

```text
상황: Meta 캠페인 세팅 완료 후 미디어믹스와 실제 세팅값 일치 여부 확인 필요
기존 방식: 엑셀과 매체 관리자 화면을 번갈아 보며 수작업 비교
AdMate 사용: Sentinel이 예산/기간/타겟/URL/상태 비교, 오류 항목만 표시
효과: 검수 시간 감소, 캠페인 사고 예방
```

### 시나리오 3. 운영 중 이상 감지

```text
상황: 캠페인 CPC가 전일 대비 급등
기존 방식: 플래너가 직접 수치 확인 후 이상 여부 판단
AdMate 사용: Sentinel이 이상 감지 → Slack 알림 → 운영자가 확인/보류/중지 선택
효과: 대응 속도 향상, 이력 자동 기록
```

### 시나리오 4. 광고주 보고용 캡처 준비

```text
상황: 캠페인 보고서에 광고 게재 화면 캡처 필요
기존 방식: 직접 화면 구성 → 캡처 → 편집 → 보고서 삽입
AdMate 사용: Lens가 매체/상품/지면별 캡처 자동 생성
효과: 캡처 업무 시간 감소, 증빙 이미지 품질 표준화
```

### 시나리오 5. 다음 캠페인 기획

```text
상황: 동일 업종의 다음 캠페인 제안 준비
기존 방식: 과거 사례 검색, 엑셀 정리, 담당자 경험 기반 보정
AdMate 사용: Foresight가 업종/목표/예산 기준 예상 성과 제공, Hermes가 기존 운영 피드백 참고
효과: 제안 준비 시간 단축, 예측 기준 표준화
```

## 10. Before / After 요약

| 업무 | Before | After |
|---|---|---|
| 정책 확인 | 매체 도움말 직접 검색 | Compass에 질문 |
| 세팅 검수 | 엑셀과 매체 화면 수작업 비교 | Sentinel 자동 비교 |
| 운영 확인 | 사람이 수시로 수치 확인 | Sentinel 실시간 이상 감지 |

| 캡처 | 직접 화면 구성 후 저장 | Lens 자동 캡처 생성 |
| 다음 기획 | 과거 사례 수동 탐색 | Foresight 예상 성과 제공 |
| 판단 기준 | 개인 경험에 의존 | Hermes에 운영 지식 축적 |

## 11. 예상 질문과 답변

### Q1. AdMate가 플래너 업무를 대체하는 건가요?

아니다. AdMate는 반복 확인과 검수, 캡처, 이상 감지를 줄여 플래너가 전략과 판단에 집중하도록 돕는 도구다.

### Q2. 알림이 너무 많이 오면 오히려 피곤하지 않나요?

Sentinel은 알림 보류, 오늘 중지, 알림 종료, 재개 기능을 제공한다. 또한 운영자 피드백을 통해 불필요한 알림을 줄이는 방향으로 고도화할 수 있다.

### Q3. 내 판단이 시스템에 반영되나요?

권한이 있는 사용자의 판단은 Hermes 학습 후보로 기록될 수 있다. 이를 통해 유사 캠페인에서 더 나은 알림 기준과 추천 기준을 만들 수 있다.

### Q4. 정책 답변은 믿을 수 있나요?

Compass는 매체 공식 가이드와 내부 지식 기반으로 답변하는 것을 목표로 한다. 향후 답변 생성 모델과 검증 모델을 분리해 정확도를 높이는 구조로 고도화한다.

### Q5. 모든 캠페인에 바로 적용되나요?

초기에는 일부 매체와 캠페인 유형을 중심으로 PoC와 베타 운영을 진행하고, 검증된 영역부터 확장하는 방식이 적합하다.

## 12. 미디어플래너 대상 발표 스토리라인

```text
1. 왜 AdMate가 필요한가
2. 캠페인 운영에서 반복 업무가 발생하는 지점
3. AdMate가 연결하는 캠페인 운영 사이클
4. Compass: 정책 확인
5. Sentinel: 사전 검수와 실시간 모니터링
6. Lens: 캡처 자동화
7. Foresight: 다음 캠페인 예측
8. Agent Core: 내 판단이 다음 기준이 되는 구조
9. 실제 업무 시나리오
10. 앞으로 함께 고도화할 방향
```

## 13. 향후 슬라이드 구성안

| Slide | 제목 | 핵심 내용 |
|---|---|---|
| 1 | AdMate가 무엇인가 | 미디어플래너의 반복 업무를 줄이는 AI 업무 파트너 |
| 2 | 현재 업무의 어려움 | 정책 확인, 세팅 검수, 캡처, 운영 확인 부담 |
| 3 | 캠페인 운영 사이클 | 기획 → 정책 → 검수 → 운영 → 캡처 → 다음 기획 |
| 4 | Compass | 정책/가이드 확인 자동화 |
| 5 | Sentinel | 캠페인 사고 방지와 실시간 이상 감지 |
| 6 | Lens | 캡처/증빙 자동화 |
| 7 | Foresight | 다음 캠페인 성과 예측 |
| 8 | Agent Core | 플래너 판단의 학습과 고도화 |
| 9 | Before / After | 반복 업무 감소 효과 |
| 10 | 베타 사용 방향 | 피드백 기반 고도화 |
