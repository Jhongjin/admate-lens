## 1. Executive Summary

AdMate는 나스미디어 데이터분석팀이 구축 중인 **AI Agent 기반 광고 운영 자동화 플랫폼**이다.

AdMate는 단순한 챗봇이나 개별 업무 자동화 도구가 아니라, 광고 캠페인의 기획, 정책 확인, 세팅 검수, 운영 모니터링, 캡처/증빙, 성과 예측, 운영자 피드백 학습까지 연결하는 통합 플랫폼이다.

AdMate의 목표는 다음과 같다.

1. 미디어플래너의 반복 업무를 줄인다.
2. 캠페인 세팅 및 운영 사고를 예방한다.
3. 광고 운영 노하우를 데이터와 지식으로 축적한다.
4. AI Agent가 실제 업무를 실행하고 학습하는 구조를 만든다.
5. 나스미디어만의 광고 운영 지능을 장기적인 경쟁력으로 만든다.

AdMate 생태계는 네 개의 플랫폼과 하나의 Agent Core로 구성된다.

| 구성 요소 | 역할 |
|---|---|
| AdMate Compass | 광고 플랫폼 정책/가이드 기반 RAG 챗봇 |
| AdMate Sentinel | 캠페인 시작 전 세팅 검수 + 집행 후 실시간 모니터링/알림 |
| AdMate Lens | 광고 게재 화면 및 보고서용 증빙 이미지 자동 생성 |
| AdMate Foresight | 과거 데이터 기반 캠페인 성과 예측 및 플래닝 지원 |
| AdMate Agent Core | Openclaw/Hermes 기반 지능·자동화·학습·실행 공통 레이어 |

AdMate는 개별 솔루션 묶음이 아니라, 광고 운영 생애주기 전체를 하나의 AI Agent loop로 연결하는 플랫폼이다.

### 슬라이드 전환 포인트

- 슬라이드 제목: **AdMate란 무엇인가**
- 핵심 메시지: AdMate는 AI Agent 기반 광고 운영 자동화 플랫폼이다.
- 시각화 아이디어: AdMate 중심에 Agent Core를 두고 Compass/Sentinel/Lens/Foresight가 연결된 생태계 다이어그램

## 2. 왜 지금 AdMate가 필요한가

광고 운영 업무는 점점 복잡해지고 있다.

매체는 다양해지고, 광고 상품은 빠르게 변화하며, 정책과 세팅 기준도 계속 바뀐다. 미디어플래너는 캠페인 기획부터 운영, 보고까지 수많은 확인 업무를 수행해야 한다.

현재 업무에서 반복적으로 발생하는 문제는 다음과 같다.

| 문제 | 설명 | 회사에 미치는 영향 |
|---|---|---|
| 정책 확인 병목 | 매체별 정책/가이드가 분산되어 있음 | 대응 지연, 정책 리스크 증가 |
| 세팅 오류 위험 | 미디어믹스와 실제 매체 세팅값 불일치 가능 | 예산/타겟/기간 오류에 따른 사고 가능 |
| 캡처 업무 부담 | 광고 게재 화면 증빙을 수작업으로 준비 | 반복 리소스 증가, 보고 지연 |
| 운영 이상 감지 지연 | 예산 소진, KPI 이상을 사람이 수시 확인 | 대응 지연, 캠페인 품질 저하 |
| 플래닝 기준 분산 | 과거 성과 기준이 체계적으로 연결되지 않음 | 제안 품질 편차, 의사결정 시간 증가 |
| 운영 지식 축적 한계 | 운영자의 판단이 구조화되어 남지 않음 | 회사 고유 운영 노하우 자산화 어려움 |

AdMate는 이러한 문제를 단일 기능으로 해결하려는 것이 아니라, 광고 운영 흐름 전체를 연결하여 해결한다.

```text
정책 확인
→ 세팅 검수
→ 운영 모니터링
→ 캡처/증빙
→ 성과 예측
→ 피드백 학습
```

이 흐름이 하나로 연결되어야 반복 업무 감소, 사고 예방, 운영 지식 자산화가 동시에 가능하다.

### 슬라이드 전환 포인트

- 슬라이드 제목: **광고 운영 업무의 현재 문제**
- 핵심 메시지: 업무는 복잡해졌지만 확인·검수·캡처·운영 판단은 여전히 수작업에 의존한다.
- 시각화 아이디어: 업무 병목 6개 아이콘 배열 또는 Before 업무 흐름도

## 3. AdMate의 전체 구조

AdMate는 광고 운영 업무를 캠페인 생애주기 관점에서 연결한다.

```text
기획
→ 정책 확인
→ 세팅 검수
→ 집행 시작
→ 운영 모니터링
→ 캡처/증빙
→ 성과 평가
→ 다음 기획 반영
```

각 단계에는 다음 플랫폼이 대응한다.

| 캠페인 단계 | 담당 플랫폼 | 역할 |
|---|---|---|
| 기획 | AdMate Foresight | 업종/목표별 예상 성과 예측 |
| 정책 확인 | AdMate Compass | 매체 정책/가이드 기반 질의응답 |
| 세팅 검수 | AdMate Sentinel | 미디어믹스와 실제 세팅값 비교 |
| 운영 모니터링 | AdMate Sentinel | 예산/KPI/상태 이상 감지 및 알림 |
| 캡처/증빙 | AdMate Lens | 광고 게재 화면 자동 캡처 |
| 학습/자동화 | AdMate Agent Core | 실행, 기록, 학습, 지능화 |

AdMate Agent Core는 이 플랫폼들을 연결하는 내부 지능/자동화 레이어다.

Agent Core는 Openclaw와 Hermes를 기반으로 한다.

- **Openclaw**: 스케줄과 조건에 따라 업무를 실행하고, 알림과 외부 시스템 연동을 처리하는 자동화 실행 엔진
- **Hermes**: AI와 사용자 이벤트를 학습해 운영 지식과 판단 기준을 축적하는 지능/메모리 엔진

외부 설명에서는 Openclaw/Hermes를 별도 제품으로 강조하기보다, AdMate Agent Core를 구성하는 내부 기술로 설명한다.

### 슬라이드 전환 포인트

- 슬라이드 제목: **AdMate 생태계 구조**
- 핵심 메시지: AdMate는 캠페인 운영 전 과정을 4개 플랫폼과 Agent Core로 연결한다.
- 시각화 아이디어: 캠페인 생애주기 원형 플로우 + 각 단계에 제품명 배치

## 4. 플랫폼별 역할

## 4.1 AdMate Compass

AdMate Compass는 광고 플랫폼 정책과 가이드의 방향을 잡아주는 Policy Intelligence Agent다.

기존 AdMate Guide의 브랜드 확장명이며, Meta, Google, Naver, Kakao, X 등 주요 광고 플랫폼의 정책 및 가이드를 RAG 기반으로 검색하고 답변한다.

### 주요 역할

- 매체별 정책/가이드 통합 관리
- 공식 문서 기반 질의응답
- 광고 집행 전 정책 리스크 확인
- 하이브리드 검색 및 Re-ranking 기반 답변 정확도 향상
- 향후 Multi-LLM Validation을 통한 할루시네이션 최소화

### 기대 효과

- 정책 확인 시간 단축
- 매체 정책 리스크 감소
- 광고주 문의 대응 속도 향상
- 사내 정책 지식 자산화

### 슬라이드 전환 포인트

- 슬라이드 제목: **Compass: 정책과 가이드의 방향을 잡다**
- 핵심 메시지: 분산된 매체 정책을 하나의 AI 질의응답 경험으로 통합한다.
- 시각화 아이디어: 여러 매체 로고 → Compass → 답변 카드

---

## 4.2 AdMate Sentinel

AdMate Sentinel은 캠페인 시작 전 세팅 오류 검수와 집행 후 실시간 모니터링을 모두 포함하는 Campaign Validation & Live Monitoring 플랫폼이다.

기존 Sentinel beta는 Sentinel의 사전 검수 영역으로 정리하고, 현재 Openclaw/Hermes 기반으로 구현한 캠페인 모니터링과 실시간 알림 기능은 Sentinel의 Live Monitoring 영역으로 포함한다.

### Sentinel의 두 영역

| 영역 | 역할 |
|---|---|
| Pre-launch Validation | 캠페인 시작 전 미디어믹스와 실제 매체 세팅값 비교 |
| Live Monitoring | 캠페인 시작 후 예산/KPI/상태 이상 감지 및 알림 |

### 주요 역할

- 미디어믹스와 매체 세팅값 비교
- 예산, 기간, 타겟, 랜딩 URL, 상태 오류 탐지
- 캠페인 시작 후 예산 소진/지연 감지
- CTR/CPC/CPM 등 KPI 이상 감지
- Slack/Email 알림
- 알림 보류/오늘 중지/알림 종료/재개
- 운영자 대응 이력 및 피드백 학습

### 기대 효과

- 캠페인 사고 예방
- 운영 중 이상 대응 시간 단축
- 미디어플래너의 수시 확인 부담 감소
- 운영자 판단 기준의 데이터화

### 슬라이드 전환 포인트

- 슬라이드 제목: **Sentinel: 캠페인 사고를 사전에 막고 실시간으로 감지하다**
- 핵심 메시지: Sentinel은 캠페인 시작 전과 시작 후를 모두 감시하는 사고 방지 플랫폼이다.
- 시각화 아이디어: Pre-launch / Live Monitoring 두 영역을 나눈 다이어그램

## 4.3 AdMate Lens

AdMate Lens는 광고 게재 화면과 보고서용 증빙 이미지를 자동 생성하는 Creative & Placement Capture Automation 솔루션이다.

기존 AdMate Capture Pro의 브랜드 확장명이며, Lens는 광고가 실제로 어떻게 노출되는지 확인하고 증빙한다는 의미를 갖는다.

### 주요 역할

- YouTube/GDN 등 광고 게재 화면 캡처
- 데스크톱/모바일 광고 UI 렌더링
- 네이티브 광고 UI 합성
- 보고서용 고해상도 증빙 이미지 생성
- 캠페인/상품/매체별 캡처 자동화

### 주의 사항

캡처 결과물 자체의 광고 미리보기/플랫폼 UI는 픽셀 매칭이 중요하므로 임의 변경하지 않는다. 디자인 통일 대상은 관리자 화면, 입력 폼, 결과 목록, 작업 이력 화면 등 운영 UI다.

### 기대 효과

- 반복 캡처 업무 감소
- 보고서 준비 시간 단축
- 광고주 제출용 증빙 품질 표준화
- 캠페인 운영 기록과 캡처 자산 연결

### 슬라이드 전환 포인트

- 슬라이드 제목: **Lens: 광고 게재 화면과 증빙을 자동으로 만들다**
- 핵심 메시지: 광고 캡처와 증빙 생성에 들어가는 반복 리소스를 줄인다.
- 시각화 아이디어: 입력 폼 → 자동 캡처 → 보고서 이미지

## 4.4 AdMate Foresight

AdMate Foresight는 과거 광고 데이터와 시장 트렌드를 기반으로 캠페인 성과를 예측하고 기획을 지원하는 Media Planning Intelligence 솔루션이다.

기존 AdMate Planner의 브랜드 확장명이며, Foresight는 캠페인 성과를 미리 내다보고 전략 방향을 제안한다는 의미를 갖는다.

### 1차 PoC 범위

Meta(Facebook/Instagram)를 1차 PoC 대상으로 한다.

선정 이유는 다음과 같다.

- 전사적으로 데이터 누적량이 가장 많음
- 캠페인 목적/지표 구조가 비교적 표준화되어 있음
- 업종·국가·소재 유형별 비교가 용이

### 주요 역할

- Meta 과거 광고 데이터 기반 예상 CPM/CPC/CTR/VTR/CPV 제공
- 업종/목표별 단가 기준 내부 자산화
- 예산 대비 예상 성과 시뮬레이션
- 시즌성/시장 트렌드 반영
- 향후 Google, Naver 등 타 매체 확장

### 기대 효과

- 캠페인 기획 단계 의사결정 시간 단축
- 제안 품질 표준화
- 업종/목표별 내부 벤치마크 자산화
- 운영 결과와 다음 기획의 연결

### 슬라이드 전환 포인트

- 슬라이드 제목: **Foresight: 다음 캠페인의 성과를 미리 예측하다**
- 핵심 메시지: 과거 성과와 시장 트렌드를 기반으로 기획 단계의 판단을 지원한다.
- 시각화 아이디어: 업종/목표/예산 입력 → 예상 CPM/CPC/CTR 출력

## 5. AdMate Agent Core

AdMate Agent Core는 AdMate 생태계의 지능과 자동화를 담당하는 공통 엔진이다.

Agent Core는 Openclaw와 Hermes를 기반으로 구성된다.

```text
Openclaw
= 업무를 실행하는 자동화 엔진

Hermes
= 지식을 축적하고 판단 기준을 고도화하는 지능/메모리 엔진

AdMate Agent Core
= Openclaw + Hermes 기반 지능형 자동화 레이어
```

### Agent Core의 역할

- 플랫폼 간 데이터와 업무 흐름 연결
- 캠페인 상태 모니터링
- 조건/스케줄 기반 자동 실행
- Slack/Email 알림 및 버튼 액션 처리
- 운영자 피드백 수집
- 운영 판단 기준 학습
- 비용/사용량/감사 로그 관리
- 향후 각 플랫폼의 Agent Tool 호출

### Agent Core가 중요한 이유

AdMate의 차별점은 개별 AI 기능이 아니라, 각 기능이 Agent Core를 통해 연결되고 학습한다는 점이다.

```text
Compass가 정책을 답한다.
Sentinel이 캠페인을 감시한다.
Lens가 캡처를 만든다.
Foresight가 다음 성과를 예측한다.
Agent Core가 이 모든 흐름을 연결하고 기록하고 학습한다.
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **AdMate Agent Core: 지능과 자동화의 중심**

- 시각화 아이디어: Agent Core 중심 허브 다이어그램

## 6. 현재 구현 수준 정리

임원 보고에서는 현재 구현 수준을 과장하지 않고, 다음 기준으로 나누어 설명하는 것이 적절하다.

| 플랫폼 | 현재 상태 | 설명 |
|---|---|---|
| Compass | 기반 구축/고도화 단계 | 멀티 플랫폼 크롤링, 본문 추출, 하이브리드 RAG 구조 보유. 정확도/검증 체계 고도화 필요 |
| Sentinel | 운영 모니터링 영역 구현 진행/완료 범위 존재 | Openclaw/Hermes 기반 캠페인 모니터링, 알림 통제, Slack action, 운영 이력, 학습 권한 구조 구현 진행. 사전 검수 영역은 별도 PoC/연계 필요 |
| Lens | 기능 구현 진행 | 광고 게재 화면 캡처/렌더링 자동화 기능 구현. 운영 콘솔화 및 Agent 연계 필요 |
| Foresight | 기획/PoC 설계 단계 | Meta 중심 과거 데이터 기반 예측 모델과 시뮬레이터 설계 필요 |
| Agent Core | 핵심 구조 구현 진행 | Openclaw/Hermes 기반 모니터링, 알림, 피드백, 학습, 사용자 권한, 감사 구조 구축 중 |

핵심 표현은 다음과 같이 정리한다.

```text
현재는 각 플랫폼의 핵심 기능과 Agent Core의 기반 구조를 구축하는 단계이며,
향후 이들을 하나의 AdMate 생태계로 연결해 통합 운영 자동화 플랫폼으로 확장할 계획입니다.
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **현재 구현 수준**
- 핵심 메시지: 일부 기능은 이미 구현되고 있으며, 전체 방향은 통합 Agent 생태계로 확장하는 것이다.
- 시각화 아이디어: 플랫폼별 상태를 `구현 진행 / PoC / 고도화 예정` 배지로 표시

## 7. 플랫폼 간 연결이 만드는 경쟁력

AdMate의 경쟁력은 각 제품 기능이 아니라, 플랫폼 간 연결에서 발생한다.

### 7.1 단일 기능과 통합 플랫폼의 차이

| 단일 기능 | AdMate 통합 구조 |
|---|---|
| 정책 챗봇 | 정책 답변이 캠페인 맥락과 연결됨 |
| 세팅 검수 | 검수 결과가 운영 모니터링과 연결됨 |
| 캡처 자동화 | 캡처 결과가 캠페인 기록과 연결됨 |
| 성과 예측 | 예측 결과가 실제 운영 결과와 다시 연결됨 |
| 알림 시스템 | 운영자 대응이 Hermes 학습으로 연결됨 |

### 7.2 회사 경쟁력 관점

AdMate가 제공하는 경쟁력은 다음과 같다.

1. **업무 효율 향상**  
   반복 확인, 검수, 캡처, 정책 검색 시간을 줄인다.

2. **운영 사고 예방**  
   캠페인 시작 전 세팅 오류와 집행 후 성과 이상을 조기에 감지한다.

3. **광고주 서비스 품질 향상**  
   정책, 검수, 캡처, 성과 예측의 응답 속도와 품질을 높인다.

4. **회사 고유 데이터 자산화**  
   운영자의 판단, 캠페인 성과, 예외 기준, 반복 이슈가 Hermes에 축적된다.

5. **AI 운영 비용 통제**  
   LLM Cost Center를 통해 비용과 ROI를 관리한다.

6. **지속적 기술 고도화**  
   Weekly Intelligence Loop를 통해 빠르게 변화하는 AI/MarTech 기술을 반영한다.

### 슬라이드 전환 포인트

- 슬라이드 제목: **AdMate가 만드는 회사 경쟁력**
- 핵심 메시지: AdMate는 업무 효율 도구를 넘어 회사 고유 광고 운영 지식을 축적하는 플랫폼이다.
- 시각화 아이디어: 효율/사고예방/서비스품질/데이터자산/비용통제 5개 축

## 8. 운영 관리 체계

AdMate는 AI를 도입하는 것에서 끝나지 않는다. AI를 지속 가능하게 운영하기 위한 관리 체계도 함께 필요하다.

## 8.1 LLM Cost Center

AdMate가 확장될수록 유료 LLM, 검색, 이미지, 비전, 임베딩, reranking, TTS 등 다양한 AI 비용이 발생한다.

LLM Cost Center는 각 플랫폼과 모델의 비용을 실시간으로 관리하는 관리자 대시보드다.

주요 관리 항목:

- 오늘/이번 주/이번 달 비용
- 플랫폼별 비용
- 모델별 비용
- 사용자별 비용
- 기능별 비용
- 요청 수
- 토큰 사용량
- 평균 요청 비용
- 예산 대비 사용률
- 비용 급증 알림

임원 보고에서는 다음 메시지로 설명한다.

```text
AI 활용을 확대하되, 비용 구조를 실시간으로 관리하고 ROI를 확인할 수 있는 운영 체계를 함께 설계하고 있습니다.
```

## 8.2 Weekly Intelligence Upgrade Loop

AI와 MarTech 기술은 빠르게 변화한다. 따라서 AdMate는 한 번 구축하고 끝나는 플랫폼이 아니라, 매주 최신 기술을 탐색하고 적용 후보를 선별하는 루틴이 필요하다.

루틴:

```text
매주 Deep Search 실행
→ 최신 AI/MarTech/광고 플랫폼 기술 변화 수집
→ AdMate 적용 가능성 평가
→ 비용 절감/성능 개선/보안 리스크 검토
→ 적용 후보 선정
→ Codex/Openclaw 작업 backlog화
```

임원 보고에서는 다음 메시지로 설명한다.

```text
데이터분석팀은 AI 기술 변화를 단순히 따라가는 것이 아니라,
매주 탐색하고 검증해 사내 플랫폼에 반영하는 체계를 구축하려고 합니다.
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **지속 가능한 AI 운영 체계**
- 핵심 메시지: 비용 통제와 기술 고도화 루틴이 있어야 AI 플랫폼은 지속 가능하다.
- 시각화 아이디어: Cost Center와 Weekly Intelligence Loop를 좌우 2분할

## 9. 권한, 보안, ISMS 관점

AdMate는 광고 운영 데이터, 캠페인 성과 데이터, 사용자 행동, 운영자 피드백, LLM 요청 로그를 다룬다.

따라서 초기 설계부터 보안과 감사 가능성을 고려해야 한다.

### 핵심 원칙

- raw campaign-level 데이터는 LLM에 직접 전달하지 않는다.
- 필요 시 익명화/집계/요약 후 전달한다.
- 민감 데이터 접근은 권한 기반으로 제한한다.
- 모든 Agent action은 audit log에 남긴다.
- 학습 반영 가능한 사용자를 제한한다.
- 외부 API key/token은 환경변수 또는 credential store에서만 관리한다.
- 브라우저에는 service role key나 ingest key를 노출하지 않는다.

### 권한 체계 방향

| 역할 | 설명 |
|---|---|
| Super Admin | 시스템 운영 및 권한 관리 |
| Admin | 관리 범위 내 사용자/캠페인 운영 |
| Reviewer | Hermes 학습에 반영 가능한 검토자 |
| User | 일반 사용자. 기본적으로 학습 신호 미반영 |

임원 보고에서는 보안 세부 구현보다 다음 메시지가 중요하다.

```text
AdMate는 AI 자동화 도입과 동시에 권한, 감사 로그, 학습 통제를 함께 설계해 ISMS 관점의 리스크를 줄이는 방향으로 구축합니다.
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **보안과 감사 가능한 AI 운영 구조**
- 핵심 메시지: AI가 행동하고 학습하기 때문에 권한·감사·학습 통제가 필수다.
- 시각화 아이디어: 권한 관리 / 감사 로그 / 학습 통제 3개 축

## 10. 장기 비전

AdMate의 1차 목표는 미디어플래너 업무 자동화와 광고 운영 지능화다.

장기적으로는 Hermes에 축적된 업무 지식과 외부 시장 정보를 결합해, 회사가 직접 구축할 가치가 있는 신규 솔루션 후보를 발굴하고 PoC 개발로 연결하는 전략 루프까지 확장할 수 있다.

이를 **Business Opportunity Discovery & Build Loop**로 정의한다.

### 장기 루프

```text
Hermes 도메인 지식 축적
→ 내부 pain point mining
→ 외부 경쟁사/솔루션 조사
→ 회사에 필요한 솔루션 후보 발굴
→ 사업성 평가
→ 사람 승인
→ 기획안 생성
→ Codex/Paperclip Builder Agent 작업화
→ PoC 구축
→ 결과 평가
→ 정식 제품화 여부 결정
```

여기서 사업성은 단순 매출만 의미하지 않는다.

- 직접 수익화 가능성
- 광고주에게 더 양질의 서비스를 제공할 가능성
- 현재 유료로 사용하는 외부 플랫폼을 대체할 가능성
- 내부 업무 시간을 절감할 가능성
- 회사 고유 데이터 자산을 활용한 차별화 가능성

임원 보고에서는 장기 비전으로 간략히 언급한다.

```text
장기적으로 AdMate는 내부 업무 데이터와 시장 정보를 기반으로 신규 솔루션 기회를 탐색하고 PoC 개발로 연결하는 전략 엔진으로 확장될 수 있습니다.
```

### 슬라이드 전환 포인트

- 슬라이드 제목: **장기 비전: 업무 자동화를 넘어 솔루션 기회 발굴로**
- 핵심 메시지: AdMate는 장기적으로 회사에 필요한 신규 AI 솔루션을 발굴하는 전략 엔진으로 확장 가능하다.
- 시각화 아이디어: 운영 지식 축적 → 기회 탐색 → PoC 개발 → 제품화 루프
