# AdMate Lens Naver Mobile Product Coverage v1

작성일: 2026-05-03

## 목적

Naver 모바일 캡처 지면을 `모바일 피드` 하나로 통합하지 않고, 성과형 디스플레이 광고 상품 단위로 분리해 AdMate Lens의 작업 목록, metadata, 결과 이미지가 실제 운영 상품 구분과 맞도록 정리한다.

## 공식 문서 기준 상품 분류

참조 문서:

- 네이버 성과형 디스플레이 광고 상품 안내: `https://ads.naver.com/help/faq/834`
- 캠페인 소재 유형 안내: `https://ads.naver.com/help/faq/899`
- 동영상 광고 소재 유형 안내: `https://ads.naver.com/help/faq/1370`

네이버 성과형 디스플레이 광고의 주요 상품군은 이미지 배너, 스마트채널, 네이티브 배너, 피드 광고, 동영상 광고, 커뮤니케이션 애드, 쇼핑 다이내믹, 쇼핑 소식, ADVoost 쇼핑 광고로 정리한다.

## 1차 구현 범위

정적 이미지 기반 모바일 증빙이 가능한 지면부터 우선 구현한다.

| Lens surface | 운영 상품명 | 구현 상태 | 비고 |
|---|---|---|---|
| `naver-smart-channel-mobile` | 스마트채널 | 1차 구현 | 기본 선택값 |
| `naver-feed-mobile` | 피드 광고 | 1차 구현 | 레거시 `naver-mobile-feed` 호환 |
| `naver-native-banner-feed` | 네이티브 배너 | 1차 구현 | 피드형 네이티브 카드 |
| `naver-image-banner-mobile` | 이미지 배너 | 1차 구현 | 모바일 배너형 |

## 작업 원칙

- 캡처 결과물은 네이티브 UI 매칭이 목적이므로 임의의 브랜드 테마를 적용하지 않는다.
- 운영자 화면과 metadata 표기는 상품 단위로 구분한다.
- 레거시 요청은 깨뜨리지 않고 신규 surface로 정규화한다.
- 후속 QA에서 실제 네이버 게재 화면 레퍼런스가 확보되면 surface별 위치, 간격, 폰트, CTA를 다시 조정한다.
