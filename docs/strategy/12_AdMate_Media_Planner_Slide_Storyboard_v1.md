# AdMate Media Planner Slide Storyboard v1

작성일: 2026-05-01  
문서 상태: 초안 v1  
대상: 미디어플래너 / 캠페인 운영 실무자  
작성 목적: AdMate를 실제 사용할 미디어플래너에게 설명하기 위한 발표 슬라이드 스토리보드, 장표별 메시지, 발표 멘트, 데모 흐름, 예상 질문과 답변을 정의한다.

---

## 1. 발표 목적

이 발표의 목적은 미디어플래너에게 AdMate를 “또 하나의 시스템”으로 소개하는 것이 아니다.

핵심은 다음 메시지를 전달하는 것이다.

```text
AdMate는 미디어플래너를 대체하는 시스템이 아니라,
정책 확인, 세팅 검수, 운영 모니터링, 캡처, 다음 캠페인 예측처럼
반복적으로 시간이 드는 업무를 줄여주는 AI 업무 파트너입니다.
```

미디어플래너 대상 발표에서는 기술 구조보다 실제 업무 변화가 중요하다.

따라서 발표는 다음 질문에 답해야 한다.

- 내 업무 중 어떤 부분이 줄어드는가?
- 어떤 캠페인 사고를 막아주는가?
- 내가 매번 확인하던 정책/세팅/캡처/성과 기준을 어떻게 도와주는가?
- Slack이나 대시보드에서 실제로 어떻게 쓰게 되는가?
- 내 판단이 시스템에 어떻게 반영되는가?

---

## 2. 발표 톤과 원칙

## 2.1 권장 톤

- “여러분을 대체하는 AI”가 아니라 “여러분의 반복 업무를 줄이는 도구”로 설명한다.
- 어려운 기술 용어보다 실제 업무 장면을 먼저 보여준다.
- 각 제품명을 기능과 함께 설명한다.
- 처음부터 Agent Core, Hermes, Openclaw를 깊게 설명하지 않는다.
- 플래너가 자주 겪는 문제를 먼저 말한 뒤, AdMate 기능을 연결한다.

## 2.2 피해야 할 표현

피해야 할 표현:

```text
AI가 알아서 캠페인을 운영합니다.
플래너 업무를 자동으로 대체합니다.
사람 판단 없이 AI가 기준을 바꿉니다.
모든 매체와 모든 캠페인에 즉시 적용됩니다.
```

권장 표현:

```text
반복 확인 업무를 줄입니다.
캠페인 사고 가능성을 낮춥니다.
플래너가 전략과 판단에 더 집중하도록 돕습니다.
권한 있는 운영자의 판단은 향후 기준 개선에 활용됩니다.
검증된 영역부터 단계적으로 적용합니다.
```

## 3. 권장 슬라이드 구성 요약

미디어플래너 대상 발표는 10~12장 구성이 적절하다.

| Slide | 제목 | 핵심 메시지 |
|---|---|---|
| 1 | AdMate가 무엇인가 | 반복 업무를 줄여주는 AI 업무 파트너 |
| 2 | 플래너 업무의 반복 지점 | 정책, 검수, 캡처, 운영 확인이 반복 리소스를 만든다 |
| 3 | AdMate 캠페인 운영 사이클 | 기획 → 정책 → 검수 → 운영 → 캡처 → 다음 기획 |
| 4 | Compass | 매체 정책과 가이드를 빠르게 확인한다 |
| 5 | Sentinel - 사전 검수 | 캠페인 시작 전 세팅 오류를 줄인다 |
| 6 | Sentinel - Live Monitoring | 집행 후 예산/KPI/상태 이상을 자동 감지한다 |
| 7 | Lens | 광고 캡처와 보고서 증빙을 자동 생성한다 |
| 8 | Foresight | 다음 캠페인의 예상 성과를 빠르게 확인한다 |
| 9 | Agent Core | 내 판단이 다음 기준으로 축적된다 |
| 10 | Before / After | 반복 업무가 어떻게 줄어드는지 한눈에 보여준다 |
| 11 | 실제 사용 시나리오 | 캠페인 하나를 등록했을 때의 흐름을 보여준다 |
| 12 | 함께 고도화할 방향 | 베타 사용과 피드백 참여 요청 |

## 4. Slide 1 — AdMate가 무엇인가

### 슬라이드 제목

```text
AdMate: 미디어플래너의 반복 업무를 줄이는 AI 업무 파트너
```

### 핵심 메시지

AdMate는 캠페인 기획부터 정책 확인, 세팅 검수, 운영 모니터링, 캡처, 다음 캠페인 예측까지 연결하는 AI Agent 기반 광고 운영 자동화 플랫폼이다.

### 화면 구성

- 좌측: AdMate 로고/타이틀
- 중앙: “기획 → 정책 → 검수 → 운영 → 캡처 → 다음 기획” 흐름
- 우측 또는 하단: Compass / Sentinel / Lens / Foresight / Agent Core 아이콘

### 발표 멘트

```text
AdMate는 미디어플래너의 업무를 대신 빼앗는 시스템이 아닙니다.
여러분이 매일 반복하는 정책 확인, 세팅 검수, 운영 수치 확인, 캡처, 다음 캠페인 기준 확인 같은 업무를 줄여주는 AI 업무 파트너입니다.

오늘은 기술 설명보다, 실제 캠페인 업무 흐름에서 AdMate가 어디를 도와주는지 중심으로 말씀드리겠습니다.
```

---

## 5. Slide 2 — 플래너 업무의 반복 지점

### 슬라이드 제목

```text
캠페인 하나를 운영하기 위해 반복되는 확인 업무
```

### 핵심 메시지

캠페인 운영에서 가장 많은 시간을 쓰는 영역은 전략 자체보다 정책 확인, 세팅 검수, 상태 확인, 캡처 같은 반복 업무다.

### 화면 구성

업무 카드 6개:

1. 정책/심사 기준 확인
2. 미디어믹스와 매체 세팅 비교
3. 예산/기간/타겟/URL 검수
4. 운영 중 KPI/예산 확인
5. 광고 게재 화면 캡처
6. 다음 캠페인 기준 확인

각 카드 아래에 “수작업”, “반복”, “실수 위험”, “시간 소요” 같은 작은 배지 배치.

### 발표 멘트

```text
플래너 업무에서 가장 피로도가 높은 부분은 매번 새롭게 전략을 짜는 일보다,
정책을 다시 찾고, 세팅값을 다시 확인하고, 운영 중 수치를 계속 보고,
캡처를 만들고, 다음 제안을 위해 과거 기준을 다시 찾는 반복 업무입니다.

AdMate는 바로 이 반복되는 확인 업무를 줄이는 것을 1차 목표로 합니다.
```

---

## 6. Slide 3 — AdMate 캠페인 운영 사이클

### 슬라이드 제목

```text
AdMate는 캠페인 운영 전 과정을 하나의 흐름으로 연결합니다
```

### 핵심 메시지

AdMate는 개별 기능이 아니라 캠페인 운영 흐름 전체를 연결한다.

### 화면 구성

원형 또는 좌→우 프로세스:

```text
Foresight
기획/예측
→ Compass
정책 확인
→ Sentinel
사전 검수
→ Sentinel
운영 감지
→ Lens
캡처/증빙
→ Hermes
학습
→ Foresight
다음 기획 반영
```

### 발표 멘트

```text
AdMate는 하나의 챗봇이나 하나의 캡처 도구가 아닙니다.
캠페인의 전 과정을 연결하는 구조입니다.

기획 단계에서는 Foresight가 예상 성과를 도와주고,
집행 전에는 Compass와 Sentinel이 정책과 세팅을 확인하고,
집행 후에는 Sentinel이 이상 신호를 감지하고,
보고 단계에서는 Lens가 캡처를 만들고,
운영자의 판단은 Hermes에 쌓여 다음 캠페인 기준에 반영됩니다.
```

---

## 7. Slide 4 — Compass: 정책 확인을 더 빠르게

### 슬라이드 제목

```text
Compass: 매체 정책과 가이드를 빠르게 확인합니다
```

### 핵심 메시지

Compass는 Meta, Google, Naver, Kakao, X의 정책과 가이드를 기반으로 캠페인 집행 전 필요한 정책 판단을 빠르게 지원한다.

### 화면 구성

- 왼쪽: 플래너 질문 예시
- 오른쪽: Compass 답변 카드
- 하단: “공식 가이드 기반”, “근거 제공”, “정책 리스크 감소” 배지

질문 예시:

```text
이 건기식 소재 문구 Meta 정책상 문제 있을까?
Google Ads 랜딩 페이지 정책 확인해줘.
Naver 검색광고 병의원 업종 제한 알려줘.
```

### 발표 멘트

```text
정책 확인은 중요하지만 매번 시간이 오래 걸립니다.
Compass는 매체별로 흩어져 있는 정책과 가이드를 기반으로 질문에 답해주는 역할을 합니다.

예를 들어 건기식 소재 문구가 가능한지, 특정 랜딩 페이지가 정책상 문제 없는지,
업종별 제한 사항이 무엇인지 빠르게 확인할 수 있습니다.
```

## 8. Slide 5 — Sentinel: 시작 전 세팅 오류를 줄이다

### 슬라이드 제목

```text
Sentinel: 캠페인 시작 전 세팅 오류를 줄입니다
```

### 핵심 메시지

Sentinel은 미디어믹스/엑셀의 기획 정보와 실제 매체 플랫폼 세팅값을 비교해 시작 전 오류를 줄인다.

### 화면 구성

좌측: 미디어믹스 데이터  
우측: 매체 플랫폼 세팅값  
중앙: Sentinel 비교 엔진  
하단: 정상 / 주의 / 오류 결과 카드

검수 항목:

- 예산
- 기간
- 타겟
- 랜딩 URL
- 캠페인 상태
- 구매 유형
- 목표

### 발표 멘트

```text
캠페인 사고는 대부분 아주 작은 세팅 오류에서 시작됩니다.
예산 단위가 다르거나, 기간이 잘못됐거나, 랜딩 URL이 잘못 들어가 있거나,
미디어믹스와 실제 매체 세팅값이 다른 경우입니다.

Sentinel은 캠페인 시작 전에 이런 항목들을 자동으로 비교해서 오류 가능성을 줄입니다.
```

---

## 9. Slide 6 — Sentinel Live Monitoring: 시작 후 이상을 감지하다

### 슬라이드 제목

```text
Sentinel: 집행 후 예산·성과·상태 이상을 감지합니다
```

### 핵심 메시지

Sentinel Live Monitoring은 캠페인 시작 후 예산 소진, KPI 급변, 캠페인 상태 이상을 감지하고 Slack/Email로 알려준다.

### 화면 구성

- 캠페인 KPI 카드
- Slack 알림 예시
- 버튼: 확인 완료 / 1시간 보류 / 오늘 중지 / 알림 종료 / 상세 로그 보기

감지 예시:

- 예산 급소진
- 예산 미소진
- CTR 급락
- CPC/CPM 급등
- 캠페인 off 상태

### 발표 멘트

```text
캠페인이 시작된 뒤에도 플래너는 계속 수치를 확인해야 합니다.
Sentinel은 예산 소진 속도나 CPC, CTR, CPM 변화처럼 중요한 이상 신호를 자동으로 감지합니다.

그리고 Slack에서 바로 확인하거나, 1시간 보류하거나, 오늘은 더 이상 알림을 받지 않도록 처리할 수 있습니다.
이 대응 이력은 자동으로 기록됩니다.
```

## 10. Slide 7 — Lens: 캡처와 증빙 자동화

### 슬라이드 제목

```text
Lens: 광고 게재 화면과 증빙 이미지를 자동으로 생성합니다
```

### 핵심 메시지

Lens는 광고 게재 화면 캡처와 보고서용 증빙 이미지 생성을 자동화해 반복 리소스를 줄인다.

### 화면 구성

3단계 흐름:

```text
캠페인/매체 정보 입력
→ 광고 게재 화면 자동 생성
→ 보고서용 이미지 저장
```

예시 영역:

- YouTube 광고 캡처
- 모바일 인피드 화면
- 네이티브 광고 화면
- 보고서 증빙 이미지

### 발표 멘트

```text
보고서나 광고주 공유를 위해 캡처를 만드는 작업도 반복 리소스가 큽니다.
Lens는 캠페인 정보와 매체 정보를 기반으로 광고 게재 화면과 증빙 이미지를 자동으로 생성하는 역할을 합니다.

수작업 캡처와 편집 시간을 줄이고, 결과물의 품질도 일정하게 맞추는 것이 목표입니다.
```

---

## 11. Slide 8 — Foresight: 다음 캠페인의 성과 예측

### 슬라이드 제목

```text
Foresight: 다음 캠페인의 예상 성과를 빠르게 확인합니다
```

### 핵심 메시지

Foresight는 과거 광고 데이터와 시장 트렌드를 기반으로 업종/목표/예산별 예상 성과를 제공한다.

### 화면 구성

입력 카드:

- 매체: Meta
- 업종: 뷰티
- 목표: 트래픽
- 예산: 3천만원
- 기간: 4주

출력 카드:

- 예상 CPM
- 예상 CPC
- 예상 CTR
- 예상 노출/클릭
- 참고 기준: 최근 최대 6개월 유사 캠페인

### 발표 멘트

```text
다음 캠페인을 제안할 때는 과거 유사 캠페인 기준을 찾고, 업종과 목표별 예상 효율을 판단해야 합니다.
Foresight는 이 과정을 도와주는 플래닝 기능입니다.

초기에는 Meta를 중심으로, 업종과 목표, 예산을 입력하면 예상 CPM, CPC, CTR 같은 기준을 제공하는 방향으로 설계합니다.
```

## 12. Slide 9 — Agent Core: 내 판단이 다음 기준이 되다

### 슬라이드 제목

```text
Agent Core: 플래너의 판단이 다음 기준으로 축적됩니다
```

### 핵심 메시지

AdMate의 차별점은 단순 자동화가 아니라 운영자 판단을 기록하고 학습해 다음 기준을 개선한다는 점이다.

### 화면 구성

흐름도:

```text
이상 알림 발생
→ 플래너 판단
→ Slack 버튼/피드백
→ Hermes 학습 후보
→ 기준 개선
→ 다음 알림 품질 향상
```

기획 단계에서는 Foresight가 예상 성과를 도와주고,
정책 확인은 Compass가 지원합니다.
캠페인 시작 전에는 Sentinel이 세팅 오류를 검수하고,
시작 후에는 Sentinel이 예산과 성과 이상을 감지합니다.
보고에 필요한 캡처는 Lens가 만들고,
운영자의 판단은 Hermes에 쌓여 다음 기준에 반영됩니다.

광고주 보고나 내부 공유를 위해 광고 게재 화면을 직접 만들고 캡처하는 일은 반복 리소스가 큽니다.
Lens는 캠페인과 매체 정보를 바탕으로 광고 게재 화면과 보고서용 이미지를 자동 생성하는 것을 목표로 합니다.

Foresight는 다음 캠페인 기획을 돕는 기능입니다.
과거 광고 데이터와 시장 트렌드를 기반으로 업종, 목표, 예산에 따른 예상 CPM, CPC, CTR 등을 제공하는 방향입니다.

AdMate에서 중요한 부분은 Agent Core입니다.
쉽게 말하면 Openclaw는 실행하고, Hermes는 기억하고 판단합니다.

AdMate는 한 번에 완성되는 시스템이 아니라, 실제 플래너의 피드백을 통해 좋아지는 플랫폼입니다.
어떤 알림이 유용한지, 어떤 정책 질문이 반복되는지, 어떤 캡처가 자주 필요한지 알려주시면 그 피드백이 다음 기능과 기준을 만드는 재료가 됩니다.

## 18. 예상 질문과 답변

## Q1. AdMate가 플래너 업무를 대체하는 건가요?

아니다. AdMate는 반복 확인과 검수, 캡처, 이상 감지를 줄여 플래너가 전략과 판단에 집중하도록 돕는 도구다.

## Q2. 알림이 너무 많이 오면 오히려 피곤하지 않나요?

Sentinel은 알림 보류, 오늘 중지, 알림 종료, 재개 기능을 제공한다. 또한 운영자 피드백을 기반으로 불필요한 알림을 줄이는 방향으로 고도화할 수 있다.

## Q3. 정책 답변을 믿을 수 있나요?

Compass는 공식 가이드와 내부 지식 기반 답변을 목표로 한다. 향후 답변 생성 모델과 검증 모델을 분리해 정확도를 높이는 구조로 고도화한다.

## Q4. 모든 캠페인에 바로 적용되나요?

초기에는 일부 매체와 캠페인 유형을 중심으로 PoC 또는 베타 운영을 진행하고, 검증된 영역부터 확장하는 방식이 적합하다.

## Q5. 내 피드백이 실제로 반영되나요?

권한과 검토 절차를 거친 피드백은 Hermes의 학습 후보가 될 수 있다. 이를 통해 유사 캠페인의 알림 기준과 추천 기준을 고도화할 수 있다.

## Q6. 캡처 결과물이 실제 매체 화면과 다르면 어떻게 하나요?

Lens는 실제 매체 화면과의 시각적 일치가 중요하다. 따라서 캡처 결과물 UI는 임의로 바꾸지 않고, 관리자 화면과 작업 관리 화면 위주로 공통 디자인을 적용한다.
