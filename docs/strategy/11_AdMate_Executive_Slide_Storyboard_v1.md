## 1. 발표 목적

이 발표의 목적은 AdMate를 단순한 사내 AI 실험이나 개별 업무 자동화가 아니라, 데이터분석팀이 구축 중인 **AI Agent 기반 광고 운영 자동화 플랫폼**으로 설명하는 것이다.

임원에게 전달해야 할 핵심은 다음 세 가지다.

1. **왜 필요한가**  
   광고 운영 업무가 복잡해지고 반복 업무와 사고 리스크가 커지고 있다.

2. **무엇을 만들고 있는가**  
   AdMate는 Compass, Sentinel, Lens, Foresight, Agent Core로 구성된 광고 운영 자동화 생태계다.

3. **회사에 어떤 경쟁력을 주는가**  
   반복 업무 절감, 캠페인 사고 예방, 광고주 서비스 품질 향상, 회사 고유 운영 지식 자산화를 가능하게 한다.

---

## 2. 권장 발표 톤

임원 보고에서는 기능을 자세히 나열하기보다, **회사 경쟁력과 조직의 진화**를 중심으로 설명한다.

권장 톤:

- 기술 과시보다 업무 문제 해결 중심
- AI 유행어보다 실제 광고 운영 효율 중심
- 개별 기능보다 생태계와 연결성 중심
- 현재 구현 수준은 과장하지 않고 단계적으로 표현
- 장기 비전은 가능성으로 제시하되, 현재 실행 범위와 분리

피해야 할 톤:

- “AI가 모든 것을 알아서 한다”는 식의 과장
- Openclaw/Hermes를 외부 제품처럼 세부 설명
- 미완성 기능을 완료된 것처럼 표현
- 기술 스택 중심 설명으로 흐르는 것

## Slide 1. AdMate란 무엇인가

### 슬라이드 제목

AdMate: AI Agent 기반 광고 운영 자동화 플랫폼

### 핵심 메시지

AdMate는 광고 캠페인의 기획, 정책 확인, 세팅 검수, 운영 모니터링, 캡처, 성과 예측, 학습까지 연결하는 AI Agent 기반 광고 운영 자동화 플랫폼이다.

### 화면 구성

- 좌측: AdMate 로고/이름과 한 줄 정의
- 중앙: Hero 메시지
- 하단: 5개 키워드
  - Plan
  - Validate
  - Monitor
  - Capture
  - Learn

### 슬라이드 문구 초안

```text
AdMate
AI Agent 기반 광고 운영 자동화 플랫폼

기획부터 운영, 검수, 캡처, 학습까지
광고 운영의 전 과정을 AI Agent가 연결합니다.
```

### 발표 멘트 초안

“AdMate는 단순한 챗봇이나 하나의 자동화 도구가 아닙니다. 광고 캠페인의 기획부터 정책 확인, 세팅 검수, 운영 모니터링, 캡처, 다음 캠페인 예측까지 연결하는 AI Agent 기반 광고 운영 자동화 플랫폼입니다.”

### 시각화 아이디어

AdMate를 중앙에 두고 `Plan - Validate - Monitor - Capture - Learn`이 원형으로 연결된 구조.

## Slide 2. 왜 지금 필요한가

### 슬라이드 제목

광고 운영 업무는 점점 복잡해지고 있습니다

### 핵심 메시지

광고 운영은 정책, 세팅, 모니터링, 보고, 예측 업무가 분산되어 있고, 여전히 많은 부분이 수작업과 개인 경험에 의존한다.

### 화면 구성

업무 병목 카드 6개:

1. 정책 확인 병목
2. 세팅 오류 위험
3. 운영 이상 감지 지연
4. 캡처/보고 리소스 부담
5. 플래닝 기준 분산
6. 운영 지식 축적 한계

### 슬라이드 문구 초안

```text
매체와 상품은 빠르게 변하지만,
광고 운영의 많은 확인 업무는 여전히 수작업에 의존하고 있습니다.
```

### 발표 멘트 초안

“미디어플래너들은 캠페인 전후로 정책을 확인하고, 세팅값을 검수하고, 운영 중 수치를 확인하고, 보고용 캡처를 만들고, 다음 제안을 준비합니다. 각각은 중요한 업무지만 반복 확인에 많은 시간이 들어가고, 작은 실수가 캠페인 사고로 이어질 수 있습니다.”

### 시각화 아이디어

Before 상태를 강조하는 흐름도: `정책 검색 -> 엑셀 비교 -> 매체 관리자 확인 -> 캡처 -> 보고서 -> 다음 제안`이 모두 분리되어 있는 형태.

## Slide 3. AdMate 생태계

### 슬라이드 제목

AdMate는 4개 플랫폼과 Agent Core로 구성됩니다

### 핵심 메시지

AdMate는 Compass, Sentinel, Lens, Foresight, Agent Core로 구성된 광고 운영 자동화 생태계다.

### 화면 구성

중앙: AdMate Agent Core  
주변 4개 제품 카드:

- Compass: Policy Intelligence
- Sentinel: Campaign Validation & Live Monitoring
- Lens: Capture Automation
- Foresight: Media Planning Intelligence

### 슬라이드 문구 초안

```text
Compass는 정책을 답합니다.
Sentinel은 캠페인 사고를 막고 감지합니다.
Lens는 캡처와 증빙을 만듭니다.
Foresight는 다음 성과를 예측합니다.
Agent Core는 이 모든 흐름을 연결하고 학습합니다.
```

### 발표 멘트 초안

“AdMate는 네 개의 제품 영역으로 구성됩니다. 정책과 가이드를 담당하는 Compass, 캠페인 검수와 모니터링을 담당하는 Sentinel, 캡처와 증빙을 담당하는 Lens, 성과 예측과 기획을 담당하는 Foresight입니다. 그리고 이들을 연결하고 학습하게 만드는 중심에 AdMate Agent Core가 있습니다.”

### 시각화 아이디어

허브 앤 스포크 구조. 중앙에 Agent Core, 외곽에 4개 플랫폼.

## Slide 4. 캠페인 생애주기 연결

### 슬라이드 제목

기획부터 다음 캠페인까지 하나의 흐름으로 연결합니다

### 핵심 메시지

AdMate는 캠페인 운영 전 과정을 단절된 업무가 아니라 하나의 연결된 생애주기로 본다.

### 화면 구성

가로 또는 원형 프로세스:

```text
기획
→ 정책 확인
→ 사전 검수
→ 집행 시작
→ 운영 모니터링
→ 캡처/보고
→ 성과 평가
→ 다음 기획 반영
```

각 단계 아래 제품 배치:

- Foresight
- Compass
- Sentinel
- Sentinel Live Monitoring
- Lens
- Hermes Learning
- Foresight

### 발표 멘트 초안

“AdMate의 핵심은 기능을 많이 만드는 것이 아니라, 캠페인 생애주기를 연결하는 것입니다. 기획 단계의 예측, 집행 전 정책 확인과 세팅 검수, 집행 후 운영 모니터링, 보고용 캡처, 그리고 다음 캠페인 기획까지 하나의 흐름으로 연결합니다.”

### 시각화 아이디어

캠페인 생애주기 원형 플로우. 마지막 `학습` 단계가 다시 `기획`으로 돌아가는 루프.

## Slide 5. 제품별 역할

### 슬라이드 제목

각 플랫폼은 광고 운영의 다른 병목을 해결합니다

### 핵심 메시지

Compass, Sentinel, Lens, Foresight는 각각 명확히 다른 역할을 담당하며, 중복이 아니라 연결을 전제로 설계된다.

### 화면 구성

4분할 카드.

| 제품 | 해결하는 문제 | 기대 효과 |
|---|---|---|
| Compass | 정책/가이드 확인 | 정책 확인 시간 단축 |
| Sentinel | 세팅 오류/운영 이상 | 캠페인 사고 예방 |
| Lens | 캡처/증빙 반복 업무 | 보고 리소스 절감 |
| Foresight | 다음 캠페인 예측 | 기획 판단 시간 단축 |

### 발표 멘트 초안

“각 플랫폼은 하나의 명확한 문제를 해결합니다. Compass는 정책 확인 시간을 줄이고, Sentinel은 캠페인 사고를 예방하며, Lens는 캡처 업무를 줄이고, Foresight는 다음 캠페인 기획 판단을 돕습니다.”

### 시각화 아이디어

제품 카드 4개 + 각 카드에 아이콘, 키워드, 기대 효과 수치 후보.

## Slide 6. AdMate Agent Core

### 슬라이드 제목

Agent Core가 AdMate를 단순 도구가 아닌 학습형 플랫폼으로 만듭니다

### 핵심 메시지

Openclaw는 실행하고, Hermes는 기억하고 판단한다. Agent Core는 이를 하나로 묶어 AdMate 전체 플랫폼이 연결되고 학습하도록 만든다.

### 화면 구성

상단: Agent Core 정의  
중앙: Openclaw / Hermes 2분할

| 내부 엔진 | 역할 |
|---|---|
| Openclaw | 스케줄·조건 기반 업무 실행, 알림, 외부 시스템 연동 |
| Hermes | AI/사용자 이벤트 학습, 운영 지식 저장, 판단 기준 고도화 |

하단: `실행 -> 기록 -> 피드백 -> 학습 -> 개선` 루프

### 발표 멘트 초안

“AdMate의 차별점은 개별 기능이 아니라 Agent Core입니다. Openclaw는 정해진 조건과 스케줄에 따라 업무를 실행하고, Hermes는 AI와 사용자의 이벤트를 학습해 운영 지식과 판단 기준을 축적합니다. 즉, 사용할수록 회사의 광고 운영 지식이 쌓이는 구조입니다.”

### 시각화 아이디어

두뇌와 손발 비유를 쓰되, 슬라이드에서는 너무 캐주얼하지 않게 `Intelligence`와 `Automation`으로 표현.

## Slide 7. 현재 구현 수준

### 슬라이드 제목

현재는 핵심 기능과 Agent Core 기반을 구축하는 단계입니다

### 핵심 메시지

일부 기능은 구현/운영 단계에 있고, 일부는 PoC/설계 단계다. 전체 방향은 하나의 AdMate 생태계로 통합하는 것이다.

### 화면 구성

제품별 상태 테이블.

| 제품 | 현재 상태 | 다음 과제 |
|---|---|---|
| Compass | 기반 구축/고도화 | 정확도·검증 체계 강화 |
| Sentinel | Live Monitoring 구현 진행/완료 범위 존재 | Pre-launch 연계 및 통합 정의 |
| Lens | 기능 구현 진행 | 운영 UI/Agent 연계 |
| Foresight | 기획/PoC 설계 | Meta 데이터 기반 예측 PoC |
| Agent Core | 핵심 구조 구현 진행 | 비용/학습/권한 고도화 |

### 발표 멘트 초안

“현재는 각 플랫폼의 핵심 기능과 Agent Core 기반 구조를 구축하는 단계입니다. 특히 Sentinel의 실시간 모니터링과 알림, 운영자 피드백 기록 구조는 이미 Openclaw/Hermes 기반으로 구현을 진행해왔고, 향후 사전 검수와 다른 플랫폼을 연결해 하나의 AdMate 생태계로 확장할 계획입니다.”

### 시각화 아이디어

상태 배지: `기획/설계`, `PoC`, `구현 진행`, `운영/고도화`.

## Slide 8. 회사 경쟁력

### 슬라이드 제목

AdMate가 만드는 경쟁력

### 핵심 메시지

AdMate는 반복 업무 자동화, 캠페인 사고 예방, 광고주 서비스 품질 향상, 회사 고유 운영 지식 자산화를 통해 경쟁력을 만든다.

### 화면 구성

5개 경쟁력 축.

1. 업무 효율 향상
2. 운영 사고 예방
3. 광고주 서비스 품질 향상
4. 회사 고유 데이터 자산화
5. AI 기반 운영 체계 내재화

### 슬라이드 문구 초안

```text
AdMate는 단순히 업무를 자동화하는 것이 아니라,
나스미디어의 광고 운영 경험과 판단을 데이터화하고
AI Agent가 실행 가능한 형태로 축적하는 플랫폼입니다.
```

### 발표 멘트 초안

“AdMate가 제공하는 가치는 단순히 시간을 줄이는 데 그치지 않습니다. 운영자의 판단, 반복되는 오류, 업종별 예외 기준, 캠페인 성과 패턴이 Hermes에 축적되면서 회사 고유의 광고 운영 지식이 만들어집니다.”

### 시각화 아이디어

5개 카드 또는 피라미드 구조: 아래는 업무 자동화, 위는 데이터 자산화/경쟁력.

## Slide 9. 지속 가능한 AI 운영 체계

### 슬라이드 제목

AI를 도입하는 것에서 끝나지 않고, 운영 체계까지 설계합니다

### 핵심 메시지

AdMate는 AI 비용 통제와 최신 기술 반영 루틴을 함께 설계해야 지속 가능한 플랫폼이 된다.

### 화면 구성

좌우 2분할.

## 4. 7분 발표 스크립트 초안

아래는 10장 기준 7분 발표 흐름이다.

### 오프닝

“오늘은 데이터분석팀에서 준비 중인 AdMate 플랫폼의 전체 방향을 공유드리겠습니다. AdMate는 단순한 챗봇이나 개별 자동화 도구가 아니라, 광고 운영 전 과정을 AI Agent 기반으로 연결하는 광고 운영 자동화 플랫폼입니다.”

### 문제 제기

“광고 운영 업무는 매체와 상품이 다양해지면서 점점 복잡해지고 있습니다. 정책 확인, 세팅값 검수, 운영 중 이상 확인, 보고용 캡처, 다음 캠페인 예측까지 많은 업무가 여전히 수작업과 개인 경험에 의존하고 있습니다. 이 과정에서 시간도 많이 소요되고, 작은 오류가 캠페인 사고로 이어질 수 있습니다.”

### AdMate 소개

“AdMate는 이 문제를 캠페인 생애주기 전체 관점에서 해결하려는 플랫폼입니다. 정책 확인은 Compass, 캠페인 검수와 운영 감지는 Sentinel, 캡처와 증빙은 Lens, 성과 예측과 기획은 Foresight가 담당합니다. 그리고 이 모든 흐름을 연결하고 학습하게 만드는 중심에 AdMate Agent Core가 있습니다.”

### Agent Core 설명

“Agent Core는 내부적으로 Openclaw와 Hermes로 구성됩니다. Openclaw는 조건과 스케줄에 따라 업무를 실행하고, Hermes는 사용자 이벤트와 운영자 판단을 학습해 지식과 판단 기준을 축적합니다. 즉, AdMate는 사용할수록 회사의 광고 운영 지식이 쌓이는 구조를 지향합니다.”

### 현재 상태

“현재는 각 플랫폼의 핵심 기능과 Agent Core 기반 구조를 구축하는 단계입니다. 특히 Sentinel의 실시간 모니터링, Slack/Email 알림, 운영자 액션 기록, Hermes 학습 권한 구조는 이미 구현을 진행해왔고, 향후 사전 검수와 다른 플랫폼 기능을 하나의 생태계로 연결할 계획입니다.”

### 회사 경쟁력

“이 플랫폼이 회사에 주는 가치는 반복 업무 자동화만이 아닙니다. 캠페인 사고를 예방하고, 광고주 대응 속도를 높이며, 운영자의 판단과 예외 기준을 데이터화해 회사 고유의 광고 운영 지식으로 축적할 수 있습니다.”

### 운영 체계

“또한 AI 플랫폼은 비용과 기술 변화 관리가 중요합니다. 앞으로는 LLM Cost Center를 통해 유료 모델과 API 사용 비용을 관리하고, Weekly Intelligence Loop를 통해 최신 AI/MarTech 기술을 매주 탐색해 적용 후보를 선정하는 운영 체계도 함께 가져가야 합니다.”

### 마무리

“AdMate의 1차 목표는 미디어플래너의 반복 업무를 줄이고 캠페인 사고를 예방하는 것입니다. 장기적으로는 Hermes에 축적된 내부 지식과 외부 시장 정보를 바탕으로, 우리 회사가 직접 구축할 가치가 있는 신규 솔루션 기회를 발굴하는 전략 엔진으로도 확장할 수 있습니다.”

## 5. 예상 질문과 답변

## Q1. 지금 실제로 어디까지 구현되어 있나요?

현재는 각 제품별로 구현 수준이 다릅니다. Sentinel의 Live Monitoring 영역과 Agent Core 기반의 알림, 운영자 action 기록, Hermes 학습 구조는 구현을 진행해왔고, Compass/Lens는 기존 기반 기능을 고도화 및 브랜드 정리하는 단계입니다. Foresight는 Meta 데이터 기반 PoC 설계 단계로 보는 것이 적절합니다.

## Q2. AdMate는 기존 업무를 대체하나요?

AdMate는 사람을 대체하는 플랫폼이 아니라, 반복 확인과 검수, 캡처, 이상 감지 업무를 줄여 플래너가 전략과 판단에 집중하도록 돕는 플랫폼입니다. 특히 중요한 판단과 위험 action은 사람 승인 기반으로 운영하는 것을 원칙으로 합니다.

## Q3. AI 비용이 많이 들지 않나요?

비용 관리를 위해 LLM Cost Center를 별도로 설계할 계획입니다. 플랫폼별, 모델별, 기능별 비용을 추적하고, 단순 작업은 경량 모델, 복잡한 판단은 고성능 모델을 쓰는 라우팅 정책을 적용할 수 있습니다.

## Q4. 보안이나 ISMS 관점에서 문제는 없나요?

초기 설계부터 권한, 감사 로그, 학습 통제를 포함합니다. raw campaign-level 데이터는 LLM에 직접 전달하지 않고, 필요 시 익명화/집계/요약 후 전달하는 방향을 원칙으로 합니다. 또한 모든 주요 action은 audit log에 남기는 구조를 지향합니다.

## Q5. 왜 데이터분석팀이 이 플랫폼을 해야 하나요?

AdMate는 단순 개발 프로젝트가 아니라 광고 운영 데이터, 성과 분석, 자동화, AI 활용이 결합된 플랫폼입니다. 데이터분석팀은 광고 운영의 데이터 구조와 기술 자동화 역량을 모두 가지고 있기 때문에, 이 플랫폼을 설계하고 회사 자산으로 확장하기에 적합합니다.
