## Decision

Result: BLOCKED FOR REPO-SAFE PNG INTAKE

The latest approved GDN PC display output remains valid as an external-only
golden baseline, but it cannot be committed under `tests/golden/images/**` yet.

The current approved capture is external-sensitive because it is based on a real
publisher capture context and can include real public article content, a real ad
placement, real brand/advertiser imagery, and production capture evidence. That
is appropriate for operator-reviewed external evidence, but not for a repo-safe
golden PNG.

## Why It Cannot Be Promoted Yet

Repo-safe golden PNG intake requires all of the following:

- exact local PNG candidate path
- matching sanitized metadata fixture
- confirmation the PNG was generated by AdMate Lens without manual image edits
- dimension match against the manifest expectation
- confirmation the image contains no advertiser, campaign, account, private URL,
  credential, personal, or private landing data
- explicit approval to commit the binary PNG as a repo golden asset

None of those repo-safe binary intake inputs are currently present in the working
tree. The repo has manifest and metadata placeholders only; there is no
committed or untracked `tests/golden/images/gdn-pc-display/golden.png` candidate.
