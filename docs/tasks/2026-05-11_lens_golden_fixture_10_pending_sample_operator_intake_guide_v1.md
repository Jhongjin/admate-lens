## Redaction And Sanitization Checklist

Before any supplied PNG or metadata can be approved for repository intake, the
operator must confirm all of the following:

- The PNG contains no advertiser name, campaign name, account name, customer
  identifier, billing identifier, workspace identifier, or private creative id.
- The PNG contains no personal data, private landing page content, private URL,
  signed URL, redirect URL, query string, token, cookie, credential, password,
  API key, session id, or internal diagnostic value.
- The PNG uses neutral test creative, generic copy, and non-sensitive public
  context suitable for long-lived repository fixtures.
- The PNG was generated by AdMate Lens and was not manually edited, retouched,
  composited, cropped, resized, redrawn, or post-processed outside the Lens
  capture output path.
- The PNG dimensions exactly match the manifest expectation for the surface.
- The metadata fixture contains only approved sanitized operational fields:
  schema version, surface id, sample state, fixture kind, captured time,
  duration, result category, diagnostics capture quality, runtime provider,
  runtime captured time, runtime duration, and required neutral
  surface-specific fields.
- The metadata fixture contains no provider request or response body, raw
  capture payload, storage key, upload URL, signed URL, browser storage dump,
  production trace, auth diagnostic, account id, campaign id, advertiser id,
  or private landing information.
- The manifest and metadata changes are reviewed separately from binary PNG
  intake and are not bundled with unrelated product code, capture execution,
  upload, storage, DB, browser session, cleanup, or environment changes.

If any item is uncertain, keep the surface as `pending-sample` or classify it as
`external-sensitive` in a separate approved gate. Do not commit uncertain binary
fixtures to `tests/golden/images/**`.

## Required Approval Wording

Use this exact approval wording for each surface that is allowed to become a
repo-safe committed golden asset:

```text
I approve committing the AdMate Lens-generated PNG for <surface> at
tests/golden/images/<surface>/golden.png as a repo-safe golden asset.

I confirm the PNG dimensions match the manifest expectation, the matching
metadata at tests/golden/metadata/<surface>.json contains only sanitized
approved fixture fields, resultCategory is ad_capture_ok, the sample was not
manually edited, and the PNG and metadata contain no advertiser, campaign,
account, personal, credential, token, signed URL, private URL, private landing,
raw provider payload, raw capture payload, browser storage, or production-only
diagnostic data.
```

Replace `<surface>` with exactly one of:

```text
demandgen-youtube-feed
kakao-bizboard
naver-smart-channel-mobile
youtube-shorts
```

Approval for one surface does not approve any other surface. Approval for a
docs-only guide does not approve binary PNG intake.
